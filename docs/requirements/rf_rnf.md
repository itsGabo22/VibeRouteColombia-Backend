# Requerimientos — Vibe Route

## Requerimientos Funcionales

| ID  | Descripción                                                        | Prioridad |
| --- | ------------------------------------------------------------------ | --------- |
| RF1 | El sistema debe permitir autenticación de administradores.          | Alta      |
| RF2 | El sistema debe permitir registrar pedidos.                         | Alta      |
| RF3 | El sistema debe geolocalizar direcciones automáticamente.           | Alta      |
| RF4 | El sistema debe permitir visualizar pedidos en un mapa.             | Alta      |
| RF5 | El sistema debe asignar pedidos a repartidores.                     | Alta      |
| RF6 | El sistema debe generar rutas optimizadas automáticamente.          | Alta      |
| RF7 | El sistema debe mostrar la ruta en el mapa.                         | Media     |
| RF8 | El sistema debe permitir modificar pedidos.                         | Media     |
| RF9 | El sistema debe sugerir correcciones de direcciones inválidas.      | Media     |

## Requerimientos No Funcionales

| ID   | Descripción                                                       | Categoría      |
| ---- | ----------------------------------------------------------------- | -------------- |
| RNF1 | El sistema debe responder en menos de 3 segundos.                  | Rendimiento    |
| RNF2 | El sistema debe ser accesible desde navegador web.                 | Accesibilidad  |
| RNF3 | El sistema debe usar arquitectura escalable.                       | Escalabilidad  |
| RNF4 | El sistema debe desplegarse en la nube.                            | Despliegue     |
| RNF5 | El sistema debe usar base de datos persistente.                    | Persistencia   |
| RNF6 | El sistema debe ser modular.                                       | Mantenibilidad |
| RNF7 | El sistema debe usar API REST.                                     | Interoperabilidad |

---

> **Nota:** Este documento se actualizará conforme se definan nuevos requerimientos durante el desarrollo del proyecto.
